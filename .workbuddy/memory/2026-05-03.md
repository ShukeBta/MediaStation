#### 2. M-Team API code=1 判断错误
- **文件**: `backend/app/subscribe/site_adapter.py`
- **原代码**: `if code_str not in ("0", "1", "200")` — 把 code=1 当成功
- **实际**: M-Team v3 API 中 code=0=成功，code=1=失败
- **修复**: 区分处理 code=0/1/401，并输出详细错误提示

#### 3. _is_title_relevant 正则表达式 bug
- **文件**: `backend/app/subscribe/service.py`
- **原代码**: `r"(?:^|\W)" + kw + r"(?:\W|$)"` — `\W` 配合 `^` 不能防止 grasshoppers 误匹配
- **修复**: 改用 `r"(?<!\w)" + kw + r"(?!\w)"` 负向前瞻/回顾断言
- **效果**: "hoppers" 不再误匹配 "Grasshoppers"，"test" 不再误匹配 "Testing"

#### 4. service.py 重复函数定义
- **文件**: `backend/app/subscribe/service.py`
- **问题**: `_parse_resolution`, `_extract_base_title`, `_is_title_relevant` 各定义了两次
- **修复**: 删除重复版本，只保留一份
