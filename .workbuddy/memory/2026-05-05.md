## 修复 main.py 启动错误

### 问题
- 后端启动时出现 `UnboundLocalError: cannot access local variable 'get_settings'`
- 原因：在 `lifespan` 函数内第 62 行重新导入了 `from app.config import get_settings`
- 这导致 Python 解释器认为 `get_settings` 是局部变量，从而在第 37 行调用时出错

### 修复
- 删除 `main.py` 第 62 行的 `from app.config import get_settings`
- 因为 `get_settings` 已在模块级别（第 17 行）导入，无需重复导入

### 结果
- ✅ 后端成功启动
- ✅ Provider Chain 成功初始化

## 发现页面 TMDB 推荐不加载修复

### 问题
- 用户报告"发现"页面不加载 TMDB 推荐内容
- 前端显示 TMDB 相关区块（流行趋势、热门电影等）为空

### 根本原因
- `discover_router.py` 直接使用 `get_settings().tmdb_api_key` 检查 TMDB 可用性
- 但用户通过设置页面配置的 TMDB API Key 存储在数据库中（通过 `ApiConfigService`）
- `get_settings()` 只读取 `.env` 文件，无法获取数据库中的配置
- 导致即使数据库有有效 API Key，`discover_router.py` 仍认为 TMDB 不可用

### 修复内容
1. **`discover_router.py`**：
   - `/sections` 端点：改用 `ApiConfigService.get_effective_config("tmdb")` 获取 TMDB 配置
   - `/feed` 端点：同样使用 `ApiConfigService` 统一配置
   - `_fetch_tmdb_section()`：新增 `tmdb_config` 参数，传递给 TMDbClient

2. **`scraper.py`**：
   - `TMDbClient.__init__()`：新增可选参数 `custom_api_key`、`custom_base_url`、`custom_image_base`、`custom_language`
   - 优先使用传入的自定义值，否则使用环境变量配置

### 验证结果
- ✅ TMDB 所有区块状态：`enabled=True`（trending、popular_movies、now_playing、popular_tv）
- ✅ `/api/discover/feed` 成功返回 TMDB 数据（每区块 12 条）
- ✅ 后端服务正常运行（PID: 4272）

### 架构问题说明
- `ApiConfigService.get_effective_config()` 采用三层优先级：数据库配置 > 环境变量 > 默认值
- 原有代码绕过了这个机制，直接读取 `get_settings()`
- 建议后续审查其他模块，确保统一使用配置服务
