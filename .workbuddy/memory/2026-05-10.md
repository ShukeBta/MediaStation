# 2026-05-10

## 安全审计修复工作

根据安全审计报告，修复了 8 项安全漏洞：

### 修复的问题：

1. **Issue #5 - Referer 泄露内网地址**
   - 文件：`backend/app/download/clients.py`
   - 修复：移除 `_try_download_torrent` 中外部请求的 `headers=self._auth_headers()`
   - 防止内网 qBittorrent 地址通过 Referer 泄露给第三方 Tracker

2. **Issue #6 - 转码任务 TOCTOU 竞态条件**
   - 文件：`backend/app/playback/transcoder.py`
   - 修复：扩大 `get_or_create_hls` 中锁的作用域，将任务注册放入锁内
   - 防止多个请求同时拉起多个 FFmpeg 进程转码同一文件

3. **Issue #7 - 危险的默认 JWT Secret**
   - 文件：`backend/app/config.py`
   - 修复：使用 `model_validator`，检测到默认弱密钥时自动生成随机密钥并警告
   - 倒逼用户在 `.env` 中配置固定密钥

4. **Issue #8 - FFmpeg 僵尸进程**
   - 文件：`backend/app/playback/transcoder.py`, `backend/app/main.py`
   - 修复：添加 `shutdown_all()` 方法，在 lifespan 关闭时清理所有 FFmpeg 子进程

5. **Issue #13 - 默认管理员重建漏洞**
   - 文件：`backend/app/main.py`, `backend/app/user/repository.py`
   - 修复：检查是否存在任何 admin 角色用户（而非仅检查 "admin" 用户名）
   - 添加 `check_admin_exists()` 方法到 UserRepository

6. **Issue #14 - SSL 证书校验被禁用**
   - 文件：`backend/app/download/clients.py`, `backend/app/config.py`
   - 修复：将 `verify=False` 改为 `verify=True`（可配置）
   - 添加 `verify_client_ssl` 配置项，仅允许内网连接时关闭校验

7. **Issue #15 - 缓存清理未检查活跃任务**
   - 文件：`backend/app/playback/transcoder.py`
   - 修复：`cleanup_cache()` 删除前检查 `self._active_jobs`，跳过正在转码的目录

8. **Issue #16 - JWT Token 缺乏吊销机制**
   - 文件：`backend/app/deps.py`
   - 修复：在 `get_current_user` 中添加 `user.is_active` 检查
   - 防止已封禁用户继续使用 token 访问
