## License Server 架构迁移完整记录

### 阶段一：Python + Node 双进程（已废弃）
- Python FastAPI 后端（port 8001）+ Node.js 管理面板（port 8002）
- Python 环境问题：`py manager` 安装的 Python 3.14 不在 PATH 中
- 进程不稳定：后台进程随 bash 会话退出而终止
- 管理面板 404：旧 FastAPI 未注册 admin_panel 路由

### 阶段二：纯 Node.js 单进程（最终方案）
- **彻底迁移**：http + better-sqlite3，零 Python 依赖
- **单端口**：8001 同时提供 API 和管理面板（内嵌 HTML）
- **兼容旧数据**：SQLite `CREATE TABLE IF NOT EXISTS` 不修改已存在的表

### Bug 修复清单

| # | 问题 | 根因 | 修复 |
|---|------|------|------|
| 1 | 创建授权码 500 错误 | 旧表 `is_revoked` 列无 DEFAULT，INSERT 未提供 | INSERT 显式包含 `is_revoked = 0` |
| 2 | ALTER TABLE 崩溃 | SQLite 不支持 `ADD COLUMN ... NOT NULL DEFAULT` | 移除所有 ALTER TABLE 迁移代码 |
| 3 | 试用授权显示"永久" | 到期时间判断未区分授权类型 | 永久类型显示"永久"，其他无到期日显示橙色"未设置到期日" |

| 5 | **中文备注乱码** | `readBody` 中 `body += chunk` 将 Buffer 按Latin-1拼接 | 改用 `Buffer.concat(chunks).toString('utf-8')` |

### 关键技术教训
- Node.js `req.on('data', chunk => body += chunk)` 中 chunk 是 Buffer，`+=` 拼接按 Latin-1 编码导致中文乱码
- 正确做法：收集 `chunks[]` 数组，最后 `Buffer.concat(chunks).toString('utf-8')`
- SQLite `CREATE TABLE IF NOT EXISTS` 不会修改已存在的表结构
- Windows 进程持久化：`powershell Start-Process -WindowStyle Hidden`
- `powershell -NoProfile` 避免环境变量 PATH 中的特殊字符导致错误
