## 刮削文件转移路径功能实现

### 需求
用户询问刮削后文件转移到哪里，需要增加自定义转移路径或选择不转移。

### 实现方案
增加 `organize.mode` 配置项，支持两个选项：
- `move_to_library`：移动到媒体库（默认，保持原有行为）
- `keep_in_place`：保留在原下载目录（不转移）

### 修改文件

#### 后端: `backend/app/download/service.py`
1. 新增 `_get_organize_setting()` 方法，用于读取整理相关设置
2. 修改 `_organize_task()` 方法，在开头检查 `organize.mode`：
   - 如果为 `keep_in_place`，直接返回空的 `OrganizeResult()`，跳过文件转移
   - 否则继续执行原有的整理逻辑

#### 前端: `frontend/src/components/settings/OrganizeScrapeTab.vue`
1. 在 `form.organize` 中增加 `mode: 'move_to_library'`
2. 在 `defaults` 中增加 `mode: 'move_to_library'`
3. 在 `loadSettings()` 中读取 `organize.mode` 配置
4. 在 `save()` 中保存 `organize.mode` 配置
5. 在模板中"文件整理规则"区块添加整理模式下拉选择框

### 配置项
- Key: `organize.mode`
- 可选值: `move_to_library` (默认), `keep_in_place`
- 向后兼容：未设置时默认 `move_to_library`

### 测试
- 后端代码语法检查通过

## TMDb 刮削失败修复（2026-05-06）

### 问题
1. "刮削失败: No results found on TMDb" - TMDb API 搜索失败
2. "保存失败: [object Object],[object Object]" - 前端错误显示问题

### 根因
1. TMDb API 搜索失败是因为 `service.py` 中使用 `get_tmdb_client()` 没有使用 `ApiConfigService` 读取数据库配置
2. 前端错误显示问题是因为后端返回的错误是数组格式，前端直接拼接导致 `[object Object]`

### 修复内容

#### 1. 后端 `backend/app/media/service.py`
修改 `scrape_item()` 方法：
- 使用 `ApiConfigService` 获取 TMDb 配置
- 创建 `TMDbClient` 时传入 `custom_api_key`, `custom_base_url` 等参数

修改 `search_tmdb()` 方法：
- 同样使用 `ApiConfigService` 获取 TMDb 配置

#### 2. 后端 `backend/app/subscribe/service.py`
修改订阅搜索中的 TMDb API 调用：
- 使用 `ApiConfigService` 获取 TMDb 配置

#### 3. 前端 `frontend/src/components/settings/OrganizeScrapeTab.vue`
修复错误处理：
```typescript
let errorMsg = '未知错误'
const detail = e.response?.data?.detail
if (Array.isArray(detail)) {
  errorMsg = detail.map((d: any) => d.msg || d.message || JSON.stringify(d)).join('; ')
} else if (typeof detail === 'string') {
  errorMsg = detail
} else if (e.message) {
  errorMsg = e.message
}
```

### 涉及文件
- `backend/app/media/service.py` - scrape_item(), search_tmdb()
- `backend/app/subscribe/service.py` - 订阅搜索中的 TMDb 调用
- `frontend/src/components/settings/OrganizeScrapeTab.vue` - 错误处理

### 状态
- 所有代码修改完成并验证语法正确
- 后端已重启
