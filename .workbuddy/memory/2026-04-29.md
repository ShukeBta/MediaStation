## 主题切换 BUG 修复 + 播放功能增强（本次会话）

### 主题切换 BUG 修复
**问题根因**：
1. `index.html` 硬编码 `class="dark"` → 页面初始加载总是深色，JS 执行后切换导致闪烁
2. `body` 使用硬编码 Tailwind 类 `bg-gray-950 text-gray-100` 而非 CSS 变量
3. CSS 中 `:root, .dark` 同时定义变量导致浅色模式被 `:root` 的深色值覆盖

**修复内容**：
- **index.html**: 移除硬编码 `class="dark"`，新增 inline `<script>` 在 HTML 解析前读取 localStorage 偏好设置 class（防止闪烁）
- **style.css**: 分离 `.dark` 和 `:root:not(.dark)` 选择器，移除 `:root, .dark` 合并写法；浅色模式 `--bg-hover` 改为 `#e2e8f0` 提高对比度
- **App.vue**: body 移除硬编码类，仅保留 `antialiased`
- **AppToast.vue**: Toast 容器和确认弹窗改用 CSS 变量背景/文字；Toast 颜色加 dark: 前缀适配浅色模式
- **AppEmpty.vue**: 硬编码 gray 类全部替换为 CSS 变量
