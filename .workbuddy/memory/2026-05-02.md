### 1. 数据库模型扩展（Phase 1）
**文件**: `backend/app/media/models.py`

新增 `MediaLibrary` 字段：
- `min_file_size`: 最小文件大小（字节），0表示不限
- `metadata_language`: 元数据语言，默认 "zh-CN"
- `adult_content`: 允许成人内容，默认 False
- `prefer_nfo`: 优先使用 NFO 元数据，默认 True
- `enable_watch`: 启用实时监控，默认 True

新增 `MediaItem` 字段：
- `hdr_format`: HDR 格式（HDR10, HDR10+, Dolby Vision, HLG）
- `audio_channels`: 音频通道数（7.1, 5.1, 2.0, stereo, mono）
- `frame_rate`: 帧率（23.976, 24, 60）
- `color_space`: 色彩空间（BT.709, BT.2020, DCI-P3）
- `bit_depth`: 位深（8, 10, 12）
- `is_duplicate`: 是否为重复项
- `duplicate_of`: 指向主条目的 ID
- `file_hash`: 文件哈希（用于重复检测）

### 2. Schema 更新（Phase 1）
**文件**: `backend/app/media/schemas.py`

- `LibraryCreate`: 新增高级设置字段（min_file_size, metadata_language, adult_content, prefer_nfo, enable_watch）
- `LibraryUpdate`: 同上
- `LibraryOut`: 同上
- `MediaItemOut`: 新增视频增强元数据字段（hdr_format, audio_channels, frame_rate, color_space, bit_depth, is_duplicate, duplicate_of）

### 3. FFprobe 增强（Phase 2）
**文件**: `backend/app/media/scanner.py`

扩展 `_ffprobe()` 方法，新增提取：
- HDR 格式：检测 HDR10、HDR10+、Dolby Vision、HLG（通过 profile 和 side_data）
- 音频通道数：从 channel_layout 或 channels 字段提取
- 帧率：从 r_frame_rate 字段计算
- 色彩空间：从 color_space 字段提取
- 位深：从 bits_per_raw_sample 字段提取

### 4. 海报发现功能（Phase 2）
**文件**: `backend/app/media/scanner.py`

新增函数：
- `is_poster_file()`: 判断文件是否为海报图片

  - 同名图片（video.mkv -> video.jpg）
  - 标准命名（poster.jpg, cover.png, folder.jpg, thumb.jpg 等）
  - 父目录查找（对于剧集）
  - 向上递归查找最多 2 层目录
- `find_all_posters()`: 查找所有海报图片

常量定义：
- `POSTER_EXTENSIONS`: {".jpg", ".jpeg", ".png", ".webp"}
- `POSTER_NAMES`: ["poster", "cover", "folder", "thumb", "fanart", "backdrop", "movie"]
