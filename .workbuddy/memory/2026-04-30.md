- **原因**: 之前是遍历所有符合条件的资源全部下载
- **修复**:
  - 新增画质优先级解析 `_parse_resolution()` 和基础标题提取 `_extract_base_title()`
  - 搜索结果按基础标题分组，每组只选一个最高画质版本
  - 画质优先级: 2160p(4K) > 1080p > 720p > 480p，HDR/DoVi 加分
  - 只下载综合评分最高的 1 个资源
  - 增加详细日志记录每个候选资源的筛选原因

#### 问题4：qBittorrent 无法下载 M-Team tracker URL 格式的种子
- **问题**: M-Team genDlToken 返回 tracker URL（`https://tracker.m-team.cc/announce?credential=xxx`），qBittorrent 无法处理
- **原因**: tracker URL 不是种子文件下载 URL，qBittorrent 的 `/torrents/add` API 无法直接使用
- **参考项目**: MoviePilot V2（https://github.com/jxxghp/MoviePilot）
- **MoviePilot 解决方案**: 使用 `qbittorrentapi` 库，支持传递种子文件二进制（`torrent_files` 参数）
- **修复**:
  1. `QBittorrentAdapter.add_torrent()` 增强：
     - 检测 URL 内容类型（Content-Type）
     - 如果是种子文件下载 URL，先下载内容再作为二进制上传
     - 对 tracker URL 等特殊格式也尝试下载并作为种子文件上传
  2. `MTeamAdapter.get_download_url()` 优化：
     - 改进日志记录
     - 正确识别 .torrent 下载 URL（通过 final_url 和 content_type）
     - 统一返回 HTTP URL 供 qBittorrent 处理

#### 支持的下载 URL 格式
| 格式 | 说明 | 处理方式 |
|------|------|----------|
| `magnet:?xt=urn:btih:...` | 标准磁力链 | 直接传递给 qBittorrent |
| `https://...torrent` | 种子下载 URL | 下载内容，上传二进制 |
| `TORRENT:<base64>` | Base64 种子 | 解码后上传二进制 |
| `https://tracker.m-team.cc/...` | M-Team tracker | 下载种子或构造磁力链 |
| `http://.../download.php?id=xxx` | NexusPHP 直链 | 直接使用 |
